## 链表操作

操作链表中元素的顺序 ，一定要建立一个虚拟头节点 ， 防止 操作头节点破坏链表结构 ， 防止断链

新建的节点用虚拟头节点赋值 ， 就是把 虚拟头节点的地址赋给 新建的节点  ， 新建节点变成了引用类型 ， 改变新建节点的指向就是改变虚拟头节点的指向 。 新建节点始终都是引用类型 ， 必须考虑要把引用的对象按照正确的顺序 链接节点



#### 思路：

建立虚拟头节点 ， 并建立一个引用节点 ， 作为操作链表段的前一段的尾节点 ， 要连接两链表的头尾 达到正确顺序 。 

可以把这个想想成两个链表在连接 ， 虚拟头节点开始就是上一个链表的尾巴节点 ， 要正确连接下一个链表的头节点。

**因为链表的头节点会变动， 所以并不能确定头节点是哪一个元素，必须用先连接头节点再交换**

**过程就是把一个链表拆成两个再连接**

```java
import java.util.List;

class Solution {
    public ListNode swapPairs(ListNode head) {
        ListNode dummp = new ListNode(0);
        dummp.next = head;         ListNode nextGroup  = end.next;//后驱节点        

        ListNode prev = dummp;

        while (prev.next != null && prev.next.next != null) {
            ListNode left = prev.next;
            ListNode right = prev.next.next;

            prev.next = right;
            left.next = right.next;
            right.next = left;

            prev = left;

        }
        return dummp.next;

    }
}
```
